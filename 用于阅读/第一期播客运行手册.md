# 第一期播客运行手册

## 基准流程

第一期默认使用“飞书转写 + MiniMax 文本决策 + 本地 ffmpeg 剪辑”。OpenAI API 默认关闭，不作为主流程依赖。

## 运行前

- 把原始音频放到 `inputs/audio/radio.m4a`。
- 从飞书导出带时间戳的转写稿，优先放到 `inputs/transcript/feishu.srt`，其次是 `feishu.vtt`。
- 把采访大纲和剪辑简报放到 `inputs/outline.md`。
- 把 `config.example.yaml` 复制为 `config.yaml`，确认路径正确。
- 设置 `MINIMAX_API_KEY` 用户环境变量。
- 安装 ffmpeg，并确认 `podcast-pipeline doctor` 通过。
- 不要求设置 `OPENAI_API_KEY`；OpenAI 只允许作为手动开启的可选复核。

## 导入飞书转写

先运行：

```powershell
podcast-pipeline import-transcript --config config.yaml
```

导入后必须生成：

```text
outputs/transcript/transcript.json
```

如果时间戳缺失、乱序、空段过多，或者总时长明显对不上 `radio.m4a`，先停止，不进入后续剪辑。

## Demo 校准

运行：

```powershell
podcast-pipeline content-map --config config.yaml
podcast-pipeline demo-edl --config config.yaml --version v1
podcast-pipeline assemble-demo --config config.yaml --version v1
```

你只需要听 `outputs/demos/demo_v1.wav`。反馈可以直接用中文描述：节奏、信息密度、主持人/嘉宾比例、故事性和自然程度。

重复生成 demo 版本，直到风格合适。被确认的 demo 是完整正片剪辑的风格样本。

## 冻结风格

只有在你明确确认某一版 demo 可以之后，才运行：

```powershell
podcast-pipeline freeze-style --config config.yaml --approved-version vN
```

冻结后的风格文件就是完整正片剪辑的依据。

## 完整正片

运行最终剪辑清单和粗剪组装：

```powershell
podcast-pipeline final-edl --config config.yaml
podcast-pipeline assemble-final --config config.yaml
```

第一期目标是自然完整优先，配置范围为 45-60 分钟。不要为了卡死分钟数牺牲语义完整度。

## 后期处理

根据系统生成的 handoff 文件使用 Cleanvoice 和 Auphonic。中文口头禅处理要保守。
